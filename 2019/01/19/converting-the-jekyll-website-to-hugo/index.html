Converting the Jekyll website to Hugo and Blogdown

January 19, 2019 R, R Markdown

Introduction

Using Jekyll as a static site generator with GitHug Pages on a Windows computer required setting up an Ubuntu environment to issue a jekyll serve command. Doing so became annoying. Another big selling point was the inclusion of R code in posts, a feature that I quickly became accostomed to after writing a couple of course websites in R Markdown and publishing them with GitHub Pages.1

\(\LaTeX{}\)

Including inline \(\LaTeX{}\) math is done by wrapping expressions in a single $. Multiline expressions are wrapped in $$. For example, $$ \begin{equation} \label{eq:some-eq} \mathbf{y} = \mathbf{X} \beta + \varepsilon, \end{equation} $$ which can be referred to as \(\eqref{eq:some-eq}\). Automatic equation numbering is accomplished by including the HTML code

<script type="text/x-mathjax-config">
MathJax.Hub.Config({
  TeX: { equationNumbers: { autoNumber: "AMS" } }
});
</script>

in a post. See http://docs.mathjax.org/en/latest/tex.html#automatic-equation-numbering.
